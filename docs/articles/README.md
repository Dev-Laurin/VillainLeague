# Feature Articles

This directory contains blog-style articles documenting the development of features in Villain League.

## Purpose

Each article provides a comprehensive, reader-friendly explanation of:
- How a feature was implemented
- Why certain technical decisions were made
- Challenges faced and solutions found
- Code examples and explanations
- Impact on gameplay

## Naming Convention

Articles should be named using the format: `YYYY-MM-DD-feature-name.md`

Examples:
- `2026-01-10-special-attack-system.md`
- `2026-01-15-character-class-system.md`
- `2026-01-20-status-effects-implementation.md`

## Required Sections

Each article should include:

1. **Title and Metadata** (Date, Author, Feature, PR link)
2. **Overview** (What was built and why)
3. **Technical Approach** (Architecture and design patterns)
4. **Implementation Details** (Step-by-step with code snippets)
5. **Challenges and Solutions** (Problems encountered and how they were solved)
6. **Testing** (How the feature was tested)
7. **Player Impact** (How this improves gameplay)
8. **Future Improvements** (Potential enhancements)

## Audience

These articles are written for:
- Developers interested in how features were built
- Contributors looking to understand the codebase
- Players curious about the technical side of the game
- Anyone wanting to learn from the development process
